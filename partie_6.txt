Regarde cette partie du code :

await fetch(SUPABASE_URL + "/report", {

Le site envoie les rapports à un endpoint /report sur le Worker. Le Worker relaie ensuite ces données à un Webhook Discord. Si le Worker ne filtre pas correctement le contenu de l'objet embed, un attaquant peut :
1. Modifier l'apparence du rapport pour tromper les admins.
2. Exfiltrer des tokens : En injectant un script ou une URL malveillante dans le champ reason du rapport, tu peux forcer l'envoi de données privées vers ton propre serveur.
Nouvelle vulnérabilité : "Heartbeat Presence" Hijacking
J'ai repéré une fonction heartbeat() dans ton code :
JavaScript
const res = await fetch(

Cette fonction utilise un on_conflict=session_id. C'est une faille d'Insecure Upsert.
